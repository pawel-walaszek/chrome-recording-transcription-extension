# Specyfikacje

Używaj tego katalogu na robocze specyfikacje implementacyjne przyszłych zmian, gdy zmiana jest większa niż bezpośredni bugfix.

Używaj numerowanych nazw plików: `NNN-krotki-tytul.md`, na przykład `001-stan-bledu-nagrywania.md`.

Zacznij od [_template.md](_template.md), a potem zostaw tylko sekcje pomocne dla konkretnej zmiany.

Po implementacji usuń nieaktualną specyfikację albo przenieś trwałe ustalenia do `docs/contracts/`.
